# 多源信息聚合接入方案调研

> **日期**：2026-04-21
> **调研目的**：寻找低成本自动化的信息源接入方式，为品猹"内容遴选"功能提供数据支撑

---

## 一、调研背景

品猹计划构建"内容遴选"功能，需要从十余个信息源自动获取高质量内容。核心思路是两层架构：
1. **发现层**：通过信息聚合平台发现"什么值得读"
2. **处理层**：品猹自有 pipeline 对原文链接做二次抓取和清洗

本次调研围绕"发现层"展开，评估各类平台的程序化接入可行性。

---

## 二、语鲸调研结果

语鲸（Yujing）是一款小众的 AI 每日资讯摘要 App（iOS/Android），主打"AI 筛选 + 短摘要推送"。

| 维度 | 评估 |
|------|------|
| 官方 API | 无 |
| RSS 输出 | 无 |
| Webhook | 无 |
| 邮件订阅 | 未知 |
| 程序化接入 | **不可行**（仅移动端 push） |
| 知名度 | 极低，主流科技媒体无报道 |

**结论：语鲸不适合作为数据源接入。**

## 五、RSSHub 方案评估（推荐）

### 5.1 项目概况

RSSHub 是一个开源 RSS 订阅源生成器（GitHub 40k+ stars），核心理念"万物皆可 RSS"，支持 1000+ 条路由，覆盖数百个中外网站。

### 5.2 品猹相关路由覆盖

| 平台 | 路由示例 | 状态 |
|------|---------|------|
| YouTube 频道 | `/youtube/channel/:id` | ✅ 支持（可能需 API Key） |
| Bilibili UP主/排行榜 | `/bilibili/user/video/:uid`, `/bilibili/ranking/:rid` | ✅ 支持 |
| 36氪 | `/36kr/newsflashes` | ✅ 支持 |
| 虎嗅 | `/huxiu/article` | ✅ 支持 |
| 少数派 | `/sspai/matrix` | ✅ 支持 |
| InfoQ | `/infoq/recommend` | ✅ 支持 |
| 知乎 | `/zhihu/hot` | ✅ 支持 |
| 微博热搜 | `/weibo/search/hot` | ✅ 支持 |
| Hacker News | `/hackernews/best` | ✅ 支持 |
| ProductHunt | `/producthunt/today` | ✅ 支持 |
| TechCrunch | `/techcrunch/news` | ✅ 支持 |
| GitHub Trending | `/github/trending/daily` | ✅ 支持 |
| 小宇宙播客 | `/xiaoyuzhou/podcast/:id` | ✅ 支持 |

**结论：品猹关注的全部平台均已被 RSSHub 覆盖。**

### 5.3 部署需求

- Docker Compose 一键部署（RSSHub + Redis）
- 最低资源：1 核 CPU / 512MB RAM
- 默认端口 1200
- 支持 JSON Feed 输出（`.json` 后缀），Python 直接解析
- 内置过滤参数（`?filter=AI&limit=10`）

### 5.4 接入品猹的架构

```
RSSHub (端口1200) ──JSON Feed──→ Celery Worker ──→ 全文抓取/LLM摘要 ──→ 数据库 ──→ API
```

### 5.5 成本

| 项目 | 成本 |
|------|------|
| 服务器增量 | 约 512MB RAM，可复用现有机器 |
| 开发工作量 | 约 2-3 天（Celery 定时拉取 + 内容入库） |
| 维护成本 | 极低，社区维护路由，docker pull 更新即可 |

## 六、补充方案：DailyHotApi

[DailyHotApi](https://github.com/imsyy/DailyHotApi)（Node.js）提供微博/知乎/百度/36氪/少数派等数十个平台的统一 REST API，支持 Docker/Vercel 一键部署。可作为 RSSHub 的补充，专门用于获取各平台热榜排名数据。

---

## 七、方案对比总结

| 方案 | 推荐度 | 覆盖面 | 成本 | 稳定性 | 理由 |
|------|--------|--------|------|--------|------|
| **自建 RSSHub** | ⭐⭐⭐⭐⭐ | 全覆盖 | 极低 | 高 | 社区活跃、JSON友好、部署轻量 |
| RSSHub + DailyHotApi | ⭐⭐⭐⭐ | 全覆盖 | 低 | 高 | 双保险，热榜+内容互补 |
| TrendRadar | ⭐⭐⭐ | 中文热榜 | 低 | 中 | 数据源依赖第三方、GPL 风险 |
| 语鲸等 App 类产品 | ⭐ | 有限 | — | — | 无法程序化接入 |

**最终结论：推荐采用自建 RSSHub 作为内容遴选的数据采集层。** 详见 `PRD-内容遴选服务.md`。
